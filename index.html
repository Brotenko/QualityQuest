Welcome to the Scrum-Meeting Protocols#

The documentation of the protocols will be written in markdown (.md) for easier viewing and reading through Github. This also allows us to create a HTML/CSS/JS output via MkDocs whenever needed.

Document structure#

The document is split up in three different sections for easier viewing and reading.

Section Content
Sprint-Reviews Protocols of the conducted sprint-reviews held with the Scrum-Master and the customer.
Scrum-Meetings Protocols of the conducted scrum-meetings held with the Scrum-Master and the team.
Miscellaneous Protocols of miscellaneous mettings conducted with people that are not directly involved with the project.


How to use MkDocs#

For full documentation visit mkdocs.org.

  • mkdocs new [dir-name] - Create a new project.
  • mkdocs serve - Start the live-reloading docs server.
  • mkdocs build - Build the documentation site.
  • mkdocs -h - Print help message and exit.
